# LENS Workbench Agent Instructions

## Available Slash Commands

This file defines slash command handlers for LENS Workbench lifecycle management.

### Feature Initialization & Lifecycle

When the user types `/init-feature <featureId>`:
- Delegate to the LENS agent with command: `bmad-lens-init-feature create <featureId>`
- Pass any additional flags like `--domain`, `--service`, `--track`

When the user types `/feature-status [<featureId>]`:
- Show the current feature status from git and feature.yaml
- Include phase, branch names, and recent activity

When the user types `/switch-feature <featureId>`:
- Switch the active feature context for subsequent commands

When the user types `/list-features [--status <status>]`:
- List all features from feature-index.yaml with their current status

### Planning & Validation

When the user types `/plan <featureId>`:
- Run the full planning pipeline: business plan → tech plan → finalized plan
- Include adversarial review at each stage
- Output: business-plan.md, tech-plan.md, finalized-plan-review.md

When the user types `/validate-plan <featureId>`:
- Validate frontmatter and planning state
- Check for required fields and consistency

When the user types `/next-action [<featureId>]`:
- Determine the one unblocked next action for the current feature
- Auto-delegate into that action (e.g., auto-launch appropriate skill)

When the user types `/batch <featureId> --target <target>`:
- Generate a two-pass batch intake for the specified planning target
- Resume an existing batch if already started

### Git & Branch Operations

When the user types `/read-git-state`:
- Read and display current git branch, commit state, and working tree status
- Show info relevant to current feature context

When the user types `/orchestrate-git <subcommand>`:
- Execute git workflows atomically (create branches, commit, push, create PRs)
- Return operation log with success/failure for each step

When the user types `/feature-yaml [read|write] [--feature-id <id>]`:
- Read or write feature.yaml files  
- Always operate atomically to prevent conflicts

### Governance & Rules

When the user types `/load-constitution`:
- Resolve and display governance rules from constitution YAML
- Show 4-level hierarchy: defaults → domain rules → service rules → feature overrides

When the user types `/apply-theme [--theme <name>]`:
- Apply visual theme to LENS output
- Use specified theme or user's default preference

### Problem & Risk Management

When the user types `/log-problem <featureId> <description> [--phase <phase>] [--category <cat>]`:
- Capture a problem or blocker in feature's problem log
- Timestamp and categorize automatically

When the user types `/resolve-problem <featureId> --problem-id <id> [--resolution <text>]`:
- Mark a logged problem as resolved
- Record resolution notes

When the user types `/list-problems <featureId> [--status open|resolved]`:
- Display all logged problems
- Filter by status if specified

When the user types `/analyze-problems <featureId>`:
- Analyze problem log with pattern detection and frequency analysis
- Suggest remediation priorities

### Pause, Resume & State

When the user types `/pause <featureId> --reason <text>`:
- Pause the feature preserving all state
- Record pause reason and timestamp

When the user types `/resume <featureId>`:
- Resume a paused feature restoring context
- Check for state conflicts before resuming

When the user types `/pause-status <featureId>`:
- Show pause state, reason, when paused, and any blockers to resuming

### Retrospective & Completion

When the user types `/generate-report <featureId>`:
- Generate a narrative retrospective report from problem log and artifacts
- Include lessons learned and metrics

When the user types `/update-insights <featureId> [--dry-run]`:
- Append lessons learned to governance insights store
- Make them available to future features

When the user types `/document-project <featureId>`:
- Run project documentation workflow before archival
- Create README, architecture docs, and reference guide

When the user types `/finalize-feature <featureId>`:
- Confirm all gates are satisfied
- Archive feature and mark as complete

When the user types `/archive-status <featureId>`:
- Check whether a feature has been archived

### Feature Restructuring

When the user types `/move-feature <featureId> --target-domain <domain> [--target-service <service>]`:
- Relocate a feature to new domain/service
- Patch all cross-references automatically

When the user types `/list-references <featureId>`:
- List all cross-references to a feature across governance repo

When the user types `/split-feature <featureId> --new-feature-id <newId> --story-ids <ids>`:
- Create a new feature from subset of stories
- Update indices and references

When the user types `/validate-split <featureId> --story-ids <ids>`:
- Validate story IDs for feasibility of splitting

When the user types `/move-stories <sourceId> --target <targetId> --story-ids <ids>`:
- Move story files from source to split feature

### Dashboard & Reporting

When the user types `/generate-dashboard [--output <path>]`:
- Generate self-contained HTML dashboard
- Show all features, status, dependencies, and timeline

When the user types `/dependency-graph`:
- Build dependency graph data for visualization

When the user types `/domain-status <domain>`:
- Show all features in a domain with their status

When the user types `/portfolio-status`:
- Show all active features across the portfolio with summary metrics

### Migration (Lens v3 → v4)

When the user types `/scan-legacy`:
- Scan for legacy LENS v3 branches
- Build migration plan with impact analysis

When the user types `/migrate-feature <featureId>`:
- Migrate single feature from v3 to v4 2-branch topology

When the user types `/check-conflicts`:
- Check for naming conflicts before running migrations

### Setup & Administration

When the user types `/setup-lens`:
- Install or update LENS module configuration and help registry
- Run in headless mode with inline values to skip prompts

When the user types `/onboard`:
- Scaffold governance repo and initial configuration
- Set up user profile and preferences

When the user types `/preflight`:
- Check all prerequisites before governance setup
- Report any missing dependencies

When the user types `/contextual-help [--phase <phase>]`:
- Show help topics filtered for current lifecycle phase

When the user types `/search-help --query <text>`:
- Search help topics by keyword

When the user types `/all-help`:
- Display all available help topics grouped by category

## Command Format

All commands follow this pattern:

```
/command-name <required-arg> [optional-arg] [--flag value]
```

Example invocations:
```
/init-feature my-feature --domain payments --service checkout --track agile
/plan my-feature
/finalize-feature my-feature
/log-problem my-feature "CI/CD pipeline timeout"
/move-feature my-feature --target-domain platform --target-service infrastructure
```
