# lens-help Prompt Audit

## Purpose Summary
- [../../../../.github/prompts/lens-help.prompt.md#L5](../../../../.github/prompts/lens-help.prompt.md#L5) is a control-repo stub, not an implementation prompt. It delegates to the release prompt at [../../../../.github/prompts/lens-help.prompt.md#L10](../../../../.github/prompts/lens-help.prompt.md#L10).
- The release prompt is also a stub and redirects execution to the Lens help skill at [../../../../lens.core/_bmad/lens-work/prompts/lens-help.prompt.md#L5](../../../../lens.core/_bmad/lens-work/prompts/lens-help.prompt.md#L5) and [../../../../lens.core/_bmad/lens-work/prompts/lens-help.prompt.md#L10](../../../../lens.core/_bmad/lens-work/prompts/lens-help.prompt.md#L10).
- Effective behavior lives in [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L10](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L10): contextual, phase-aware help with progressive disclosure, backed by actual capability metadata rather than hardcoded prose. The skill explicitly says help must be derived from real capabilities and sourced from [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L12](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L12), [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L33](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L33), and [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L47](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L47).
- Operationally, the prompt is a lifecycle navigation surface: contextual help, keyword search, and full-topic listing are the three declared routes in [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L53](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L53) and [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L54](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L54), with script-backed implementations documented in [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/references/contextual-help.md#L14](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/references/contextual-help.md#L14) and [../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/references/search-help.md#L3](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/references/search-help.md#L3).

## Confidence
- Overall: High.
- Rationale: The prompt chain is explicit and unambiguous, and the module, module-help, skill contract, and help-topic registry all consistently describe Lens help as a contextual routing surface.
- Residual uncertainty: Medium for exact runtime rendering details because prompt behavior is mostly delegated to skill/reference/script contracts rather than contained in prompt-local instructions.

## Gaps
- Double-stub indirection lowers prompt-local auditability. Neither the control prompt nor the release prompt contains acceptance criteria, fallback behavior, or lifecycle logic; all meaningful behavior is externalized to the skill and its references.
- Discoverability is split across multiple surfaces that can drift independently: global BMAD registries ([../../../../lens.core/_bmad/_config/skill-manifest.csv#L7](../../../../lens.core/_bmad/_config/skill-manifest.csv#L7), [../../../../lens.core/_bmad/_config/bmad-help.csv#L41](../../../../lens.core/_bmad/_config/bmad-help.csv#L41)), Lens module registration ([../../../../lens.core/_bmad/lens-work/module.yaml#L70](../../../../lens.core/_bmad/lens-work/module.yaml#L70), [../../../../lens.core/_bmad/lens-work/module-help.csv#L17](../../../../lens.core/_bmad/lens-work/module-help.csv#L17)), and the actual `/help` topic registry ([../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L33](../../../../lens.core/_bmad/lens-work/skills/bmad-lens-help/SKILL.md#L33)).
- There is real vocabulary drift between the help skill and the lifecycle contract.
