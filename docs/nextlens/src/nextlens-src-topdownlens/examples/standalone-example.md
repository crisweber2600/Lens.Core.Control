# Standalone Bottom-Up Feature Example

## Need

An operator needs a tiny feature that exports one planning artifact to Markdown for personal review.

## Entity Record

- Stable ID: `feature.export_single_artifact_markdown`
- Status: active
- Current path: `docs/features/export-single-artifact-markdown/feature.yaml`
- Known parent capability: none
- Known product area: none

## Decision

The feature remains standalone. It has no repeated pressure, no shared workflow with another feature, and no evidence that a durable capability exists.

## Evidence

- One operator request.
- One produced artifact.
- No repeated Salmon signals.

## Result

No capability, product area, domain, or service is promoted. The feature can remain independent forever.
