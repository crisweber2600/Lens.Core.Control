# TL-12 - Dogfooding Acceptance Run

## Goal

Run TopDownLens on itself end-to-end as the closure gate for this feature. Capture system thesis -> selected feature -> BMAD packet -> implementation -> verification, all using the contracts defined in TL-1, TL-4, TL-8, and TL-9.

## Scope

- Capture system thesis for `nextlens`.
- Discover product areas and select one.
- Walk down to one selected feature using TL-2 walkthrough format.
- Produce a BMAD packet using TL-4 schema.
- Hand to BMAD (simulated or real) and record the resulting artifacts list.
- Verify topology and constitution layering match TL-8 and TL-9.
- Record at least one Salmon signal (using TL-6) for any inconsistency found.

## Acceptance

- A complete top-down walkthrough is recorded in `docs/nextlens/src/nextlens-src-topdownlens/dogfood/`.
- The walkthrough produces exactly one selected feature and one BMAD packet.
- All entities in the chain have stable IDs and are referenced by ID.
- Topology is verified in `nextlens-control` and `nextlens-governance`.
- **First-run acceptance is RELAXED:** verification of `nextlens-release` is explicitly skipped because the release repo does not yet exist. This is recorded in the dogfood report with a deferred-verification note and a Salmon signal targeting the future release-repo creation.
- A dogfood report is committed at `docs/nextlens/src/nextlens-src-topdownlens/dogfood/report.md` with verdict pass / pass-with-warnings / fail.
