# TL-10 - Bugfix Flow (Lens-Core-Bugfix Pattern)

## Goal

Document a structured bug-fix lifecycle for self-hosted LENS that routes from Salmon signals (TL-6) through the topology defined in TL-8.

## Scope

- Bug intake from Salmon signals.
- Triage and severity routing (info / low -> backlog; medium -> next sprint; high / blocking -> immediate or correct-course).
- Fix branch model under control repo.
- Verification surface (target repo only).
- Closure criteria (signal resolved, evidence recorded).

## Acceptance

- Each severity has a documented routing path with owner role.
- Fix work occurs only on control plan branches and target dev branches; no direct writes to governance feature folders or release paths.
- Closure requires a resolved Salmon signal with evidence_refs.
- One worked example bugfix is recorded in `examples/bugfix-example.md`.
