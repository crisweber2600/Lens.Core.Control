# TL-11 - GitHub Actions Pipelines

## Goal

Provide CI workflows for the self-hosting topology: contract validation, governance publish gate, and a regression+doctor pipeline.

## Scope

- Contract validation workflow (runs on PRs to control plan branches).
- Governance publish gate workflow (runs on publish-to-governance).
- Regression-and-doctor workflow (runs on dev branches; consumes TL-7 output if available).
- Workflow placement under `.github/workflows/` in the appropriate repo.

## Acceptance

- Each workflow has a documented trigger, success criteria, and failure routing.
- Workflows do not write outside their assigned repo.
- Governance publish gate refuses to merge if metadata or required artifacts are missing.
- Workflows degrade gracefully if TL-5 / TL-7 are deferred (no hard dependency on those CLIs for the first increment).

## Files To Produce

- `.github/workflows/contract-validation.yml` (control repo).
- `.github/workflows/governance-publish-gate.yml` (governance repo or control repo depending on trigger source).
- `.github/workflows/regression-and-doctor.yml` (target repo).
- `docs/nextlens/src/nextlens-src-topdownlens/guides/ci-pipelines.md` (operator notes).

## Notes For Dev

- CLI/config story. Workflow YAML lives in repos under the topology defined in TL-8.
- For the first increment, regression-and-doctor may run a placeholder until TL-5 / TL-7 land.
