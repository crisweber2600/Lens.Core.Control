# Story 2.1 — Bug Discovery & Feature Generation

## Context

The `lens-bug-fixer --fix-all-new` command discovers all New bugs and creates a single
express-track feature for the batch. This is the entry point for the entire fix orchestration
pipeline.

**featureId formula:** `lens-dev-new-codebase-bugfix-{ms-timestamp}-{random4hex}`
- Generated by `init-feature-ops.py create` with slug `bugfix-{ms-timestamp}-{random4hex}`
- Millisecond timestamp prevents slug collisions (BS-2 mitigation from review)

**Feature-index sync:** Handled natively by `init-feature-ops.py create`. No separate
`--update-feature-index` flag needed — that flag does not exist in the current CLI surface
(verified 2026-05-03). Story ACs reflect this resolved finding (A3).

**Phase 1-2 boundary:** Feature MUST be created before any status mutation. If feature
creation fails, all bugs remain in New and no commits are made.

Depends on: Stories 1.3, 1.2, 1.1 (Sprint 1 complete).

## Tasks

1. Create `lens.core/_bmad/lens-work/scripts/bug-fixer-ops.py` with `discover-new` command:
   - Scan `governance_repo/bugs/New/` for all `.md` files
   - Parse frontmatter from each file; return JSON array of `{ slug, path, title }`
   - Apply scope guard before scanning
2. Add Phase 1-2 logic to `bmad-lens-bug-fixer` SKILL.md conductor:
   - If 0 bugs: print "0 bugs discovered. Queue is clean." and exit 0 (A6)
   - If N bugs: group into batch; generate featureId timestamp
3. Delegate feature creation to `bmad-lens-init-feature` skill:
   - Invoke `init-feature-ops.py create` with slug, domain=lens-dev, service=new-codebase, track=express
   - Verify `init-feature-ops.py` flag surface before implementation (flags may differ from tech-plan spec)
   - Confirm `index_updated` is `true` in response; abort batch if false
4. Handle Phase 2 failure: if feature creation fails, stop; all bugs remain in New; report error.
5. Create `lens.core/_bmad/lens-work/skills/bmad-lens-bug-fixer/SKILL.md` via **bmad-module-builder**.
6. Create `lens.core/_bmad/lens-work/prompts/lens-bug-fixer.prompt.md` (release prompt) via **bmad-workflow-builder**.
7. Create `.github/prompts/lens-bug-fixer.prompt.md` (stub).
8. Commit with message: `[dev:2.1] lens-dev-new-codebase-bugbash — bug discovery and feature generation`.

## Acceptance Criteria

- [ ] `bug-fixer-ops.py discover-new` returns JSON list of all New bugs with slug, path, title
- [ ] 0 bugs: workflow exits cleanly with "0 bugs discovered. Queue is clean." message; no mutations (A6)
- [ ] N bugs: batch formed; featureId generated as `lens-dev-new-codebase-bugfix-{ms-timestamp}-{random4hex}`
- [ ] Feature created via `init-feature-ops.py create` with track=express; team includes current_runner
- [ ] `feature-index.yaml` updated natively by init-feature-ops.py (no `--update-feature-index` flag call)
- [ ] Feature creation failure: all bugs remain in New; no mutations; explicit error report
- [ ] `.github/prompts/lens-bug-fixer.prompt.md` stub exists and invokes light-preflight.py

## Implementation Notes

- `bug-fixer-ops.py discover-new` output: `{ "bugs": [{ "slug": str, "path": str, "title": str }], "count": int }`
- `bug-fixer-ops.py` does NOT call `init-feature-ops.py` directly — SKILL.md conductor delegates to bmad-lens-init-feature as a separate skill step
- Verify `init-feature-ops.py` available flags with `--help` before implementation
