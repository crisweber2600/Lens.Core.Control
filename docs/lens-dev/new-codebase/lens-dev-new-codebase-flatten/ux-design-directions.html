Design Directions

Directional mockups for a no-UI, command-first experience. These are planning references for message hierarchy and status framing.

Direction 1: Audit Console
Structured-firstStrict states

Dense logs, strong blocker formatting, highly explicit phase and gate indicators.

Direction 2: Guided Terminal
BalancedAction prompts

Compact command output with clear "what next" affordances and reduced cognitive load.

Direction 3: Minimal Deterministic
Flat-mode friendlyLow noise

Minimal verbosity, strict consistency in status lines, branch context shown only when relevant.

Direction 4: Policy Lens
Governance visibilityMode transparency

Each output block starts with resolved mode + policy source and then command outcome.

Direction 5: Delivery Momentum
Flow-firstPhase velocity

Highlights handoff readiness, blockers, and confidence signals with concise summaries.

Direction 6: Inclusive Ops
AccessibilityReadable by default

Simple language, high-contrast status cues, keyboard-safe interaction assumptions.
