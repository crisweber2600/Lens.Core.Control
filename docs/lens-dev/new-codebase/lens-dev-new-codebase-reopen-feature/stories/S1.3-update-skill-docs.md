# S1.3 — Update `lens-feature-yaml` SKILL.md with Reopen Documentation

## Context

This story adds a documented `reopen` command entry to the `lens-feature-yaml` SKILL.md so that operators know how to invoke reopen and understand its preconditions, defaults, and accepted deferrals.

## Target File

```
TargetProjects/lens-dev/new-codebase/lens.core.src/_bmad/lens-work/skills/lens-feature-yaml/SKILL.md
```

## Required Content

Add a `reopen` command section that includes:

1. **Description:** One sentence — restores a terminal feature to active planning.
2. **Precondition:** Feature must be in a terminal state (`phase: complete` or `status: archived`).
3. **Arguments table:**

   | Arg | Required | Default | Description |
   |---|---|---|---|
   | `--feature-id` | Yes | — | Feature identifier |
   | `--governance-repo` | Yes | — | Path to governance repo root |
   | `--to-phase` | No | `expressplan` | Phase to restore to (must be non-terminal) |
   | `--actor` | No | `system` | Audit label recorded in phase_transitions |

4. **Example invocation:**
   ```bash
   uv run --with pyyaml python \
     TargetProjects/lens-dev/new-codebase/lens.core.src/_bmad/lens-work/skills/lens-feature-yaml/scripts/feature-yaml-ops.py \
     reopen \
     --feature-id my-feature \
     --governance-repo ./TargetProjects/lens/lens-governance \
     --to-phase expressplan \
     --actor alice
   ```
   > **Note:** `uv run --script` is not supported for this file due to import path issues when executed from a temp-copy location. Use `uv run --with pyyaml python <path>/feature-yaml-ops.py` instead.

5. **Deferral note:** A dedicated `/lens-reopen` conductor is not included in this implementation slice and is explicitly deferred to a future feature.

## Acceptance Criteria

- SKILL.md `reopen` section is present and accurate relative to the S1.1 implementation
- All four args are documented with correct defaults
- Deferral note is present
- No stale or contradictory information in surrounding SKILL.md content

## Definition of Done

- [ ] `reopen` section added to SKILL.md
- [ ] Args table complete and accurate
- [ ] Example invocation matches implemented command contract
- [ ] Deferral note included
