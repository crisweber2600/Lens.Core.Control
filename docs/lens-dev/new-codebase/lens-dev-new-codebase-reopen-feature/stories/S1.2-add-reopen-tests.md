# S1.2 — Add Automated Tests for the `reopen` Subcommand

## Context

This story adds pytest coverage for the `reopen` subcommand implemented in S1.1. Tests use a temp-dir fake `feature.yaml` fixture to remain independent of any live archived feature in the repo.

## Target File

```
TargetProjects/lens-dev/new-codebase/lens.core.src/_bmad/lens-work/skills/lens-feature-yaml/scripts/tests/test-feature-yaml-ops.py
```

## Test Cases

### TC-1: Success — archived feature reopens to expressplan

**Setup:** Create a temp-dir with `feature.yaml` where `phase: complete`, `status: archived`, `completed_at` is set, `track: express`.

**Action:** Call `reopen --feature-id test-feature --to-phase expressplan --actor test`.

**Assertions:**
- `feature.yaml` `phase` == `expressplan`
- `feature.yaml` `status` == `active`
- `feature.yaml` has no `completed_at` key
- `feature.yaml` `phase_transitions` list has an entry with `to: expressplan` and `actor: test`
- Return code == 0

### TC-2: Guard — non-terminal feature rejected

**Setup:** Create a temp-dir with `feature.yaml` where `phase: dev`, `status: active`.

**Action:** Call `reopen --feature-id test-feature --to-phase expressplan`.

**Assertions:**
- Return code != 0
- Output JSON contains `"reason": "reopen_not_allowed"`

### TC-3: Index sync assertion

**Setup:** Create a temp-dir with `feature.yaml` (terminal state) and a minimal `feature-index.yaml` that includes the test feature with `status: archived`.

**Action:** Call `reopen --feature-id test-feature --to-phase expressplan`.

**Assertions:**
- `feature-index.yaml` entry for the test feature has `status: active`
- Output JSON contains `"index_synced": true`

## Test Fixture Pattern

```python
@pytest.fixture
def terminal_feature(tmp_path):
    feature_yaml = tmp_path / "feature.yaml"
    feature_yaml.write_text(yaml.dump({
        "feature_id": "test-feature",
        "phase": "complete",
        "status": "archived",
        "completed_at": "2026-01-01T00:00:00Z",
        "track": "express",
        "phase_transitions": []
    }))
    return tmp_path
```

## Acceptance Criteria

- All three test cases pass via `uv run python -m pytest`
- Tests use temp-dir fixtures; no live governance repo access required
- Tests are deterministic and idempotent
