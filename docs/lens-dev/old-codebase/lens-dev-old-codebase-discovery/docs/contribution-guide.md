# lens.core.src — Contribution Guide

**Date:** 2026-04-21
**Feature:** lens-dev-old-codebase-discovery
**Module Version:** 4.0.0

## Purpose

This guide summarizes the contributor rules for `lens.core.src` — source-of-truth boundaries, validation expectations, and release impact.

## Source-of-Truth Rules

- Module behavior changes go in `_bmad/lens-work/`.
- Keep `_bmad/lens-work/module.yaml`, registry files, and installer expectations consistent.
- The downstream `lens.core/` in a control repo is a pulled release artifact — do not edit it directly.

## Where to Change What

### Module behavior

Change `_bmad/lens-work/` when modifying lifecycle behavior, skills, prompts, contracts, templates, or scripts.

### Documentation

- Use `docs/` (this folder) for project-level discovery documentation (feature lens-dev-old-codebase-discovery).
- Use `_bmad/lens-work/docs/` for module-level technical reference material.

## Validation Expectations

Before pushing, validate the subsystem you changed.

### Default checks

```bash
uv run _bmad/lens-work/scripts/validate-lens-bmad-registry.py
uv run --with pytest pytest _bmad/lens-work/scripts/tests/test-install.py -q
```

### For broader script changes

```bash
uv run --with pytest pytest _bmad/lens-work/scripts/tests -q
```

### After release promotion

From the outer control-repo root:

```bash
git -C lens.core pull
uv run ./lens.core/_bmad/lens-work/scripts/preflight.py
```

## Packaging Constraints

- Keep executable product files confined to approved script and installer locations.
- Changes to `_module-installer/installer.js`, registry files, and module manifests are coupled — review them together.

## Review Checklist

- Does the change belong in `_bmad/lens-work/`?
- If registry or manifest data changed, did you run the relevant validation?
- If documentation changed, did you update the right doc layer?

## Commit and Promotion Flow

1. Edit `lens.core.src`.
2. Validate the changed surface.
3. Commit and push.
4. Wait for the promotion workflow to succeed.
5. Pull the downstream `lens.core/` copy.
6. Run workspace preflight.
